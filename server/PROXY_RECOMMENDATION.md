# 代理配置推荐

## 测试结果

根据测试结果：
- ✅ **HTTP代理（127.0.0.1:33210）**：连接成功
- ❌ **SOCKS5代理（127.0.0.1:33211）**：连接失败（协议错误）

## 推荐配置

### 使用HTTP代理（推荐）

```
启用代理: ✅
代理类型: HTTP
代理地址: 127.0.0.1:33210
需要认证: ❌
```

**原因**：
- HTTP代理已验证可用
- 连接稳定
- 配置简单

### 为什么不使用SOCKS5？

SOCKS5代理（33211端口）测试失败，错误提示：
- "SOCKS5协议错误：代理服务器可能不是SOCKS5代理"
- 可能的原因：
  1. 该端口不是真正的SOCKS5代理
  2. 代理服务器配置问题
  3. 协议不兼容

## 配置步骤

1. **进入系统配置**
   - 量化管理 -> 系统配置 -> 代理配置

2. **选择HTTP代理**
   - 代理类型：**HTTP**
   - 代理地址：**127.0.0.1:33210**

3. **测试连接**
   - 点击"测试连接"
   - 应该显示"连接成功"

4. **启用并保存**
   - 打开"启用代理"开关
   - 点击"保存配置"

## 注意事项

1. **全局配置**：此配置为项目全局配置，所有用户共享
2. **已验证可用**：HTTP代理（33210）已测试成功，可直接使用
3. **如果HTTP不工作**：可以尝试SOCKS5，但需要先确认SOCKS5代理服务器配置正确

## 后续优化

如果将来需要支持SOCKS5：
1. 确认SOCKS5代理服务器配置正确
2. 验证SOCKS5端口（33211）是否真正支持SOCKS5协议
3. 测试SOCKS5连接是否正常
