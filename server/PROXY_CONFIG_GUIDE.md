# 代理配置指南

## 根据您的环境变量配置

根据您提供的环境变量：
```bash
export https_proxy=http://127.0.0.1:33210
export http_proxy=http://127.0.0.1:33210
export all_proxy=socks5://127.0.0.1:33211
```

### 推荐配置

#### 方案1：使用SOCKS5代理（推荐）
```
启用代理: ✅
代理类型: SOCKS5
代理地址: 127.0.0.1:33211
需要认证: ❌
```

**优点**：
- SOCKS5支持更多协议（TCP/UDP）
- 性能通常更好
- 支持更多应用场景

#### 方案2：使用HTTP代理
```
启用代理: ✅
代理类型: HTTP
代理地址: 127.0.0.1:33210
需要认证: ❌
```

**优点**：
- 兼容性好
- 配置简单

## 配置步骤

1. **进入系统配置**
   - 量化管理 -> 系统配置 -> 代理配置

2. **选择代理类型**
   - 推荐使用 **SOCKS5**（端口33211）
   - 或使用 **HTTP**（端口33210）

3. **填写代理地址**
   - SOCKS5: `127.0.0.1:33211`
   - HTTP: `127.0.0.1:33210`

4. **测试连接**
   - 点击"测试连接"按钮
   - 确认连接成功

5. **启用代理**
   - 打开"启用代理"开关
   - 保存配置

## 常见问题

### Q: 应该使用SOCKS5还是HTTP？
A: 推荐使用SOCKS5（端口33211），因为：
- 支持更多协议
- 性能更好
- 如果SOCKS5不工作，再尝试HTTP（端口33210）

### Q: 为什么测试失败？
A: 检查：
1. 代理服务器是否正在运行
2. 端口是否正确（33211或33210）
3. 代理类型是否匹配

### Q: 如何验证代理是否可用？
A: 使用命令行测试：
```powershell
# 测试SOCKS5
curl --socks5 127.0.0.1:33211 https://api.ipify.org?format=text

# 测试HTTP
curl --proxy http://127.0.0.1:33210 https://api.ipify.org?format=text
```

## 注意事项

1. **全局配置**：此配置为项目全局配置，所有用户共享
2. **代理服务器**：确保代理服务器正在运行
3. **端口匹配**：确保配置的端口与代理服务器实际端口一致
4. **测试建议**：配置后务必先测试连接，确认无误后再启用
