# 波动率配置页面404错误修复说明

## 问题分析

路由已正确注册，但可能后端服务需要重启才能生效。

## 已完成的修复

1. ✅ 路由已注册：`server/internal/router/admin.go` 第74行
   ```go
   trading.VolatilityConfig, // Trading 波动率配置
   ```

2. ✅ API定义正确：`server/api/admin/trading/volatility_config.go`
   - 路径：`/volatility/config/list`
   - 方法：GET

3. ✅ 控制器已实现：`server/internal/controller/admin/trading/volatility_config.go`

4. ✅ 服务已注册：`server/internal/logic/toogo/volatility_config.go`
   ```go
   func init() {
       service.RegisterToogoVolatilityConfig(NewToogoVolatilityConfig())
   }
   ```

## 解决方案

### 步骤1：重启后端服务

```bash
# 停止当前服务（如果正在运行）
# 然后重新编译并启动
cd server
go build -o hotgo_v2
./hotgo_v2
```

### 步骤2：验证路由是否注册成功

启动服务后，检查日志中是否有路由注册信息，或者访问：
- Swagger文档：`http://localhost:8000/swagger`
- 查看是否有 `/admin/volatility/config/list` 路由

### 步骤3：检查API路径

前端请求路径：`/volatility/config/list`
实际后端路径：`/admin/volatility/config/list`（自动加上 `/admin` 前缀）

如果前端有自定义的baseURL配置，需要确保路径正确。

## 常见问题

### 问题1：404 Not Found
**原因**：后端服务未重启，新路由未生效
**解决**：重启后端服务

### 问题2：服务未注册
**原因**：`init()` 函数未执行
**解决**：确保 `volatility_config.go` 文件被正确导入

### 问题3：路由路径不匹配
**原因**：API路径配置错误
**解决**：检查 `api/admin/trading/volatility_config.go` 中的路径定义
